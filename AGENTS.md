# Agent memory

## Learned User Preferences

- Git: when pushing changes to the remote repository, push to **`main`** (merge or work on `main` as the default integration branch unless the user explicitly asks for a different branch).
- Prefer slow, smooth UI animations; when the user reports "chunky," "fast without smoothness," or "weird," use spring physics or longer durations with ease-in-out instead of short duration + cubic ease.
- For header or chip collapse flows, include all relevant tabs (e.g. "All") in the animated exit so they fade or collapse together; collapse the group's maxWidth to 0 so hidden elements do not affect layout.
- When an animated element (e.g. plus icon with translateX) escapes its container despite overflow-hidden, prefer removing the transform over adding more overflow or wrapper fixes.
- Collection chips and dashboard segment chips: active state uses `bg-foreground text-background`; inactive state uses `text-foreground` with no background fill — not `bg-primary text-primary-foreground`.
- When applying a styling change to one chip variant, apply it consistently to all related variants (All tab, Trash tab, CollectionChip, etc.) without waiting to be asked.
- All clickable UI elements (buttons, interactive chips, clickable wrappers, menu triggers) should explicitly use `cursor-pointer`; disabled actions should use `disabled:cursor-not-allowed` so they clearly look non-interactive.
- The overall UI should lean into a premium, modern, minimal dark-luxury aesthetic (cinematic glass layers, subtle gradients, restrained motion) rather than noisy or heavy visual treatments.
- In settings sections that poll (e.g. Friends), show the loading skeleton only on initial load, not during background refreshes.

## Learned Workspace Facts

- **Env and dev**: Server loads `.env` from `apps/server` via `load-env.ts`. Web client env (e.g. `NEXT_PUBLIC_SERVER_URL`) belongs in `apps/web/.env`; restart the Next dev server after changing it. When running the API from the repo root, Bun `--hot` watches `packages/auth`, `packages/env`, and `packages/db`. **`NEXT_PUBLIC_WEB_APP_URL`** is the main app origin (default `https://cura.page`); use it for links from profile subdomains (e.g. “Back to dashboard”) so users leave `*.cura.page` for `/dashboard`.
- Better Auth: avoid `authClient.useSession()` in SignInForm and SignUpForm on this stack (Next 16, React 19, Turbopack); use `getSession()` or render the form without session check.
- When the Next.js app and API use different origins (e.g. separate Vercel projects), session cookies are set on the API host; RSC requests to the web origin do not include those cookies. Do not rely on server-side `authClient.getSession({ fetchOptions: { headers: await headers() } })` to see the API session—use client-side session checks (e.g. `AuthGate`, `getSession` in the browser) and authenticated `fetch` to the API with `credentials: "include"` (and set `createAuthClient` / server `trustedOrigins` + `CORS_ORIGIN` accordingly).
- **Safari / ITP / cross-browser auth**: If the web app and API are on different **sites** (e.g. `cura.page` vs `*.vercel.app`), Safari may block session cookies even with `SameSite=None; Secure`. **Fix:** proxy the API on the **same origin** as the web app. **`/api/auth/*`** is handled by **`apps/web/src/app/api/auth/[[...path]]/route.ts`** using **`src/lib/upstream-api-proxy.ts`** (forwards to **`KURA_API_UPSTREAM`**, not an external `next.config` rewrite). **`/_kura/*`** is rewritten **internally** in **`next.config.ts`** to **`/api/kura-proxy/*`** (App Router cannot expose `app/_kura` as a URL; external rewrites to the API can 404 if **`KURA_API_UPSTREAM`** was missing at build). The shared proxy strips **`content-encoding`**, **`content-length`**, and **`transfer-encoding`** from upstream responses when Node `fetch` has already decompressed the body (avoids **`net::ERR_CONTENT_DECODING_FAILED`** on `/api/auth/get-session` and similar). **`createAuthClient`** uses **`NEXT_PUBLIC_WEB_APP_URL`** (origin only, no `/_kura` path): Better Auth’s URL joining would otherwise send `/api/auth` to `https://cura.page/api/auth` and **drop** a `/_kura` prefix from `baseURL`. **`NEXT_PUBLIC_SERVER_URL`** stays `https://<web-host>/_kura` for Eden treaty + non-auth fetches. **`BETTER_AUTH_URL`** on the API = **web origin** `https://cura.page` (where `/api/auth` is exposed), not `…/_kura`. **`CORS_ORIGIN`**: comma-separated + `https://*.cura.page` (see `packages/env/src/cors-origins.ts`). Optional **`AUTH_COOKIE_DOMAIN`**=`cura.page` for **`crossSubDomainCookies`**. **`src/proxy.ts`** skips rewriting **`/_kura`** and **`/api/auth`** on profile hosts.
- **Production domains (this deployment):** Web **`https://cura.page`**, API **`https://kura-server.vercel.app`**. **Web:** `KURA_API_UPSTREAM`=`https://kura-server.vercel.app`, `NEXT_PUBLIC_WEB_APP_URL`=`https://cura.page`, `NEXT_PUBLIC_SERVER_URL`=`https://cura.page/_kura`. **API:** `BETTER_AUTH_URL`=`https://cura.page`; `CORS_ORIGIN`=`https://cura.page,https://*.cura.page`; optional `AUTH_COOKIE_DOMAIN`=`cura.page`. Extension / direct API clients may still use `https://kura-server.vercel.app`.
- TanStack Form: always pass an explicit `selector` prop to `form.Subscribe`; omitting it causes "selector is not a function" at runtime. Elysia routes in `apps/server`: Vercel runs TypeScript separately from the bundler (`tsdown`); request `body` is often still typed `unknown` under strict `tsc` even when `t.Object` validates at runtime—define small interfaces matching each schema and cast `body` in handlers so Vercel’s `tsc` step passes.
- DB schema lives in `packages/db/src/schema/`; push schema changes with `bun drizzle-kit push` from `packages/db/`. R2 media storage uses bucket `kura-media`; credentials (`R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_PUBLIC_URL`) live in `apps/server/.env`; the S3-compatible SDK endpoint is `https://{R2_ACCOUNT_ID}.r2.cloudflarestorage.com`; `R2_PUBLIC_URL` must be the public-facing `pub-*.r2.dev` (or custom domain) URL, not the API endpoint.
- Base UI: `ContextMenu.SubmenuTrigger` must be a direct child of `ContextMenu.SubmenuRoot` — placing it as a sibling of `ContextMenu.Submenu` throws a runtime error.
- When using `event.currentTarget` in a React setState updater (e.g. checkbox `onChange`), read the value (e.g. `checked`) into a local variable at the start of the handler and use that variable inside the updater; otherwise `currentTarget` may be null when the updater runs.
- Public profile pages use the dedicated domain **`cura.page`**: canonical URLs **`https://<username>.cura.page`**; in-app links use **`getPublicProfileUrl`** from `apps/web/src/lib/public-profile-url.ts`. **`src/proxy.ts`** rewrites `*.cura.page` (and `*.localhost` in dev) to `app/[username]/`, and (unless `NEXT_PUBLIC_PROFILE_PATH_REDIRECT=false` or `VERCEL_ENV=preview`) issues a **308** from `/:username` on the main host to the profile subdomain. Env: **`NEXT_PUBLIC_PROFILE_DOMAIN`** (default `cura.page`). With **`typedRoutes`**, use a plain **`<a href={...}>`** for those HTTPS profile URLs, not **`next/link` `Link`** (external origins are not valid route types).
- Browser extension (`apps/extension`): API base defaults to `http://localhost:3000` in dev and `https://app.kura.so` in production builds; set `WXT_PUBLIC_API_URL` to match `NEXT_PUBLIC_SERVER_URL` when the API is on another host (e.g. `kura-server.vercel.app`). Web app links default to `https://cura.page` in production; set `WXT_PUBLIC_WEB_URL` to override. Optional override: `chrome.storage.local` key `kura_api_base`. Popup appearance is stored under `kura_popup_appearance` in `chrome.storage.local` (open via popup gear → `runtime.getURL("/appearance.html")`; `entrypoints/appearance/` avoids `options_ui`). Includes theme, radius (inner controls only — Chrome popup shell is square), density, and toggles `showCurrentPage` / `showCollectionPicker` / `showRecent`.
- Polling: followed-collection bookmarks and collection metadata about every 5s; public profile tab content about every 5s and profile header (avatar, banner, counts) about every 10s; Settings → Friends about every 5s for incoming/outgoing requests without reload.
